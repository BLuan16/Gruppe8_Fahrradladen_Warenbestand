Benotete Projekte

Hinweise:

  • Die Noten ersetzen eine Klassenarbeit in Programmierung und eine Klassenarbeit in Softwaretechnik.
  • Die Benotung findet anhand der angegebenen Kriterien statt.
  • Das Projekt bezieht sich auf die bereits bekannten Projektthemen.
  • Aufgabe ist die Implementierung einer GUI mit Datenbankanbindung mithilfe von Bindings und Entity Framework.
  • Orientieren Sie Sich bei der Implementierung an der Planung, die Sie im Unterricht von Herrn Lukas erstellt haben.
  • Die Aufgabe wird in der Gruppe bearbeitet.
  • Die Teamarbeit sollte aus den Git-Commits erkennbar sein. Falls hier Besonderheiten auftreten (z. B. Pair Programming und nur eine Team-Mitglied commited den Code), informieren Sie mich bitte darüber.
  • Falls einzelne Gruppenmitglieder ausfallen, nehmen sie möglichst schnell Kontakt mit mir auf.
  • Abgabetermin ist Montag, der 22. Februar.

Erwarteter Umfang

  • Mindestens zwei GUI-Programme (Ansichten) für verschiedene Anforderungen.
  • Die Ansichten können miteinander verbunden sein, müssen es aber nicht.
  • Es ist eine Navigation in Datensätzen möglich.
  • Datensätze können neu angelegt, geändert und gelöscht werden.
  • Es findet mindestens ein Binding an einen Aufzählungstyp statt.
  • Die Datenbank enthält mindestens drei Tabellen, die per Fremdschlüsselbeziehungen verbunden sind.

Kriterien für Benotung - Programmierung

  • Funktionalität. (60 Punkte)

    • Funktionalität des Programms ist ausreichend (siehe erwarteter Umfang)
    • Es treten keine offensichtlichen Fehler auf.
    • Nutzereingaben werden überprüft.

  • Entwurf der Oberfläche. (15 Punkte)

    • Die Oberflächen haben ein strukturiertes Layout.

  • Qualität des Codes. (15 Punkte)

    • Code ist gut strukturiert.
    • Methoden und Variablen haben sinnvolle Namen.
    • Code ist ansprechend formatiert.

  • Eine über den Unterrichtsthemen hinausgehende Funktionalität. (10 Punkte)
    Beispiele:

    • Besondere Formatierung/Gestaltung in WPF
    • Einbinden externer Bibliotheken
    • Grafische Zusammenfassung

Kriterien für Benotung - Softwaretechnik

  • Entwurf und Umsetzung der Datenbank. (35 Punkte)

    • ER-Diagramm (10 Punkte)
    • Umsetzung in SQL-Skript welches die Datenbank anlegt. Code-First ist auch möglich. Dann bitte kurzer Hinweis. (15 Punkte)
    • Einfügen von Beispieldaten per SQL oder C#-Code/ (10 Punkte)

  • Verwendung des Repositories: (20 Punkte)

    • Änderungen werden in Commits dem Repository hinzugefügt.
    • Mehrere Commits im Repositories, welche die Entwicklung nachvollziehbar machen.
    • Ein Feature wird als Branch mit anschließendem Merge hinzugefügt.

  • Code-Review (15 Punkte)

    • Es wird ein Code-Review dokumentiert
    • Nutzen Sie dazu die Pull-Request-Funktion des GitHub-Repositories

  • Anforderungen als Issues im GitHub-Repository. (20 Punkte)

    • Mindestens zwei Anforderungen werden als Issue im GitHub-Repository angelegt und später mit einem Commit verbunden.

  • Beschreibung des Projekts als README.md mit folgendem Umfang. (10 Punkte)

    • Projekttitel
    • Kurze Beschreibung, wie das Programm zu nutzen ist.
    • Mindestens zwei Screenshots (jeweils ein Screenshot pro Ansicht).
    • Auflistung der Teammitglieder mit ihren GitHub-Logins. Statt des Namens können auch nur die Initialen verwendet werden.
